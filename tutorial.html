Tutorial:

Part 1: Requirements

  • A browser that supports javascript
  • Any text editor
  • The JQuery javascript library
  • The Jasmine javascript library
  • The Handlebars javascript library

Part 2: Class Responsibility Planning

Model:

  1. Player.js
      Responsibilities:
    • Keep track of player score
    • Know winning score values
    • Know if this player is the winner
    • Check if the player has won the game following their turn
  2. Square.js
      Responsibilities:
    • Know how many points this square is worth
    • Know if this square occupied
    • Know which symbol is occupying this square
  3. Board.js
      Responsibilities:
    • Store a list of all the squares
    • Know the value of each square on the board
    • Mark a square and update the players score if successful
  4. Game.js
      Responsibilities:
    • Keep track of whose turn it is
    • Allow a player to take a turn
    • Check to see if the game has been won
    • Announce the winner of the game

Controller:

  1. gameController.js
      Responsibilities:
    • Update the html code to display correct markers
    • Create a new game
    • Control program flow
    • Announce the winner of the game
  2. publishSubscribe.js
      Responsibilities:
    • Extend the jquery library
    • Allow classes to subscribe to an event
    • Allow classes to unsubscribe to an event
    • Allow classes to publish events of a specific type

Part 3: Configuring the Jasmine test suite

  1. Download the Jasmine library
  2. Create a new test directory for your project and copy the Jasmine library to that folder
  3. Open SpecRunner.html
    • Include your models
    • Include your test classes
  4. Create test classes for all of your model classes
    • Use ModelNameSpec as the naming convention
    • Be sure to test:
      • Squares cannot be used twice
      • Storing the correct player marker for a square
      • Updating player score
      • Turn order
      • Win conditions
    Example Test Case Setup: 
        describe("when testing MyClass", function () {
        "use strict";

        //test class declaration
        var aClass;

        //init test class
        beforeEach(function () {
            aClass = new MyClass();
        });

        //test cases here
        it("should test this method", function () {
            expect(aClass.method()).toBe(EXPECTED_VALUE);
        });
    });
  5. Open SpecRunner.html in your browser to verify test cases pass

Part 4: index.html

  1. Create a new file called index.html in your root directory
  2. Link your style sheet
  3. Use script tags to link the following js libraries
    • jquery.min.js
    • handlebars-v1.3.0.js
    • model/Game.js
    • model/Board.js
    • model/Square.js
    • model/Player.js
    • controller/publishSubscribe.js
  4. Add handlebar template code as follows
    • Create a new script tag and set id="board-template" type="text/x-handlebars-template" inside the tag
    • Inside the script tags create a simple table with id="tictactoe" with only one row and one column
    • Add {{#each board}} in front of the opening tr tag. Add {{/each}} after the closing tr tag
    • Add {{#each this}} in front of the opening td tag. Add {{/each}} after the closing td tag
    • Set the td id="{{this}}"
    • The "{{#each collectionName}}" tags tell handlebars to repeat the dom listed for each member of the collection specified
    • The template is rendered when gameController calls the compile method

  6. Add markup to the body section to draw game board

Part 6: Implement Publish Subscribe Model

  1. Declare var o = $({}); to extend jquery library
  2. Use o.on.apply(o, arguments); to create subscribe method
  3. Use o.off.apply(o, arguments); to create un-subscribe method
  4. Use o.trigger.apply(o, arguments); to create publish method
  5. Check publishSubscribe.js to see my implementation

Part 7: Add game control logic to gameController

  1. Create board namespace using var BOARD_NAMESPACE = BOARD_NAMESPACE || {}
  2. Create handleMarkSquare method to handle player turn input
  3. Create handleClick method to handle the player interacting with the view DOM
  4. Create addClickHandlers to assign the handlers to their DOM elements
  5. Create compile method for Handlebars template
  6. Subscribe to the markSquare event
